Demoloaders™: Impact on publishers

A good demo game has proved over time to be a powerful tool to both sell and promote videogames

  • Each install of a demo-game represents a chance to convert that player to a customer
  • Each install represents a chance to have an evangelist talking about your game- even if they can't make a purchase then

However, as games have become more complicated, and launch timing more critical they have become more challenging to implement:

  • Authoring an effective demo-game requires skill, testing and time
  • Cutting down a game to make a demo-game without breaking the game has become more involved
  • The risk to the main game launch of delays due to the demo-game authoriing has become more severe

Tangentix Demoloaders are different:

  • They are authored automatically, so there is no launch delay or resource required
  • The behaviour of the demo-game can be controlled remotely based on real player behaviour and real-time analytics
  • The player already has the full game assets, so they don't have to download the game at purchase- minimizing purchase friction

Demoloaders™: Process & Technology

Demoloaders are created automatically from a finsihed game- no access is required to the original game code which remains unchanged.

Tangentix compression means the Demoloader is typically 3x smaller than the original game.

The Tangentix Demoloader Sandbox™ protects the content and allows the behaviour and functionality of the demo-game to be controlled from the web.

When purchased, the Demoloader Sandbox™ is removed to leave the full game in place.

Demoloaders™: Frequently Asked Questions

How long can you play on a Demoloader™?

Through the cloud-management system the publisher controls the amount and timing of free play according to the data on conversion the system provides. It's even possible to spread access to the demo over a number of days if this results in a better result.

How long does it take to make a Demoloader™?

Our systems now cover most of the major game engines and we're adding more constantly. For all the ones that are completed in takes our servers about 1-4 hours to complete the conversion

How big will a Demoloader™ be?

Typically a Demoloader™ will be around 1/3rd of the size of the original game. You can see some examples here.

How can publishers engage with players using a Demoloader™?

During gameplay our engine monitors the game activity and, at appropriate times, shows "overlays" over the paused game. These can be any web-based content, pages, video, surveys, animation etc. As all this is delivered from the cloud it can be modified instantly to suit the publishers needs.

What changes are needed to the game?

None. We take the completed gold-disk master and turn it into a Demoloader™ without changing a single byte of the program code. We need no source code access.

Does it work with Steam?

It works well with Steam. Both demo and full game can be delivered via Steam and by a process of "Adoption" by the Steam client the demo can be converted to a full game.
